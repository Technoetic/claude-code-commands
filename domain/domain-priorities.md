# /domain-priorities — 본 시스템 도메인 지식 우선순위 매트릭스

본 명령어는 `https://<your-app>.fly.dev/` <your-domain> SaaS의 **N 도메인 결손 영역**을 자동 점검하여 **우선순위 매트릭스** + **가장 시급한 지식부터 리스트업**한 md를 `vault/reports/domain-priorities-<date>.md`로 영속화한다.

`/domain-knowledge`가 도메인별 풀 영속화 (9 파일) 도구라면 본 명령어는 **횡단 우선순위 1 파일** 도구.

## 사용법

```
/domain-priorities                      # N 도메인 전체 우선순위 매트릭스
/domain-priorities --top 10             # 상위 10건만
/domain-priorities --domain saju face   # 특정 도메인 한정
/domain-priorities --refresh            # 기존 md 덮어쓰기
```

## 우선순위 산출 5 차원 (정량)

각 결손 영역에 다음 점수 부여 (0~5점):

| 차원 | 의미 | 점수 기준 |
|---|---|---|
| **① 사용자 위해** | ADR-006 자문 거절 정신상 사용자 인생 위해 정도 | 0=일반·5=결혼/이혼/의료 단정 위험 |
| **② 라이브 트래픽 영향** | 본 시스템 실 호출 빈도 + LLM 인용 빈도 | 0=호출 X·5=오늘의 운세 매 호출 |
| **③ 구현 비용** | 본 AI 단독 진행 가능성 + 추정 시간 | 0=불가능·5=즉시 (1h) |
| **④ 학술 출처 신뢰도** | WebFetch 검증 가능·KCI·ISBN | 0=가짜·5=공인 학파 |
| **⑤ ADR 위반 위험** | ADR-002·006·010 위반 가능성 | 0=무관·5=명백 위반 |

**우선순위 점수 = (① × 3) + (② × 3) + (④ × 2) + (⑤ × 2) - ③** (단, ③은 비용이므로 - 부호)
**최대 점수**: 73점

→ 점수 높을수록 우선 처리. 점수 0 이하 → DEFER.

## 행동 절차

### 단계 0 — 결손 영역 자동 수집

다음 3 출처에서 결손 영역 자동 추출:

1. **vault/decisions/ADR-*.md** frontmatter `한계 (정직)` 절
2. **engine/divination/<domain>/** 명시 TODO·placeholder·미구현 마커
3. **본 세션 누적 ⚠ 결손 보고** (라이브 평가 결과 등)

자동 수집 후 중복 제거 + 도메인 분류.

### 단계 1 — 5 차원 자동 점수화

각 결손 영역에 메인 컨텍스트 (Opus 4.7) 단독 점수화:

- ADR-006 위반 위험 점검 (결혼·의료·금융 단정 가능?)
- 본 시스템 트래픽 영향 점검 (만월 아씨 오늘의 운세에 영향?)
- 본 AI 단독 진행 가능성 (라이브 검증·외부 결단·사업 결단)
- WebFetch 학술 출처 검증
- ADR-002·006·010 위반 명시 위험

### 단계 2 — 우선순위 매트릭스 생성

```markdown
## 본 시스템 도메인 지식 우선순위 (2026-05-21 기준)

### 🔴 즉시 진행 (점수 ≥ 50)
| 순위 | 도메인 | 결손 | 점수 | 본 AI 단독? |
|---|---|---|---|---|
| 1 | dream | LLM 길몽 어휘 차단 100% (Flash Lite 한계) | 58 | ⚠ Flash 업그레이드 의존 |
| 2 | saju | 양인·괴강·백호 추가 신살 | 53 | ✅ |

### 🟡 단기 진행 (점수 30~49)
...

### 🟢 운영 데이터 후 (점수 10~29)
...

### ⏸ 사용자 결단 보류 (점수 0~9)
...
```

### 단계 3 — md 영속화

`vault/reports/domain-priorities-<YYYY-MM-DD>.md` 작성:

```yaml
---
type: domain_priorities
generated: 2026-05-21
generated_by: /domain-priorities
total_gaps: 23
immediate: 3
short_term: 8
deferred: 12
related_modules:
  - engine/divination/*/
  - vault/decisions/*.md
methodology: 5_dimension_scoring
---
```

본문 6 절 구조:
1. **우선순위 매트릭스** (🔴 즉시·🟡 단기·🟢 운영 후·⏸ 보류)
2. **상위 5건 정밀 분석** (각 결손별 ADR 위반·트래픽 영향·진행 방안)
3. **본 AI 단독 진행 가능 영역** (즉시 착수 가능 목록)
4. **사용자 결단 영역 분리** (🔵 사업·법무·인프라·학술 구매)
5. **운영 데이터 의존 영역** (실 트래픽 누적 후 측정)
6. **다음 단계 추천** (사용자에게 진행 신호 요청)

### 단계 4 — 사용자 보고


## 안전장치

### 오추정 차단

본 명령어 추출 전 의무:
- engine/divination/<domain>/ 실 모듈 풀 Read·Glob·Grep
- vault/decisions/ADR-*.md frontmatter 직접 읽기
- 본 세션 ⚠ 보고 검토 (대화 컨텍스트)
- 추정 "결손" 금지 — 실 코드·실 ADR 명시 의무

### 가짜 학술 출처 차단

④ 학술 출처 신뢰도 점수 부여 시 WebFetch 라이브 검증:
- KCI 논문 URL 실재 확인
- ISBN 교보문고·알라딘 검색
- 검증 실패 → 점수 0 + 가짜 인용 명시

### 사용자 결단 영역 명시

🔵 사업 단계 (UI·가격·결제·마케팅·법무) + 학술 구매·외부 API 키는 본 AI 단독 진행 X → "사용자 결단 보류" 영역으로 분리.

### ADR-002·006·010 정합 검증

각 결손이 본 시스템 ADR에 위반 위험 명시:
- ADR-002 학파 단정 강요?
- ADR-006 자문 거절 정신상 사용자 인생 위해?
- ADR-010 사실성 분리상 가짜 인용 가능?

## ADR 정합

| ADR | 정합 |
|---|---|
| ADR-002 학파 회피 | ✅ 다학파 병행 결손 점수화 |
| ADR-006 자문 거절 | ✅ 사용자 위해 차원 (① 차원 핵심) |
| ADR-010 사실성 분리 | ✅ 학술 출처 검증 (④ 차원) |
| ADR-015 옵션 병행 | ✅ 옵션 A 디폴트 + 옵션 B 결손 별도 점수 |

## 활용 시나리오

### 시나리오 A — 개발 우선순위 결단
사용자가 본 시스템 다음 단계 결정 시:
```
/domain-priorities --top 5
```
→ 상위 5건 분석 + 본 AI 단독 진행 가능 영역 즉시 신호 가능.

### 시나리오 B — 결손 영역만 추출
```
/domain-priorities --gaps-only
```
→ 결손 영역만 리스트 + 사업 결단 영역 분리.

### 시나리오 C — 특정 도메인 정밀 분석
```
/domain-priorities --domain dream face palm
```
→ 3 도메인 결손만 우선순위 매트릭스.

## 비용

본 명령어는 메인 컨텍스트 (Opus 4.7) 단독 실행. 외부 학술 검증 시 WebFetch. 1회 호출 평균 ~$0.12 (출력 토큰 약 25K).

## /domain-knowledge와의 차이

| 항목 | /domain-knowledge | /domain-priorities |
|---|---|---|
| 출력 | N 도메인 풀 md (9 파일) | 횡단 우선순위 1 파일 |
| 용도 | 도메인 지식 영속화 (참조용) | 결손 우선순위 결단 (의사결정용) |
| 빈도 | 신규 ADR 작성 시 | 개발 단계 변경 시 |
| 사용자 결단 영역 | 명시 | 명시 + 점수화 |
| 본 AI 단독 진행 신호 | 결손 명시 | 우선순위 + 즉시 진행 가능 영역 |

## 출력 파일 위치

- `vault/reports/domain-priorities-<YYYY-MM-DD>.md` — 일자별 영속화
- `vault/reports/INDEX.md` — 갱신 (선택)

## 주의

- 본 명령어는 **합리적 우선순위만 산출**. 사업 결단 영역은 별도 분리
- 모든 점수는 본 명령어 호출 시점 기준 — 시간 경과 시 재호출 권장
- ADR-002·006·010·015 정합 의무 유지
- 사용자 출력 면책은 본 명령어 자체에는 X (메타 도구)
